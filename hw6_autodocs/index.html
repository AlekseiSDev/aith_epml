    Требования

    1. Техническая документация (2 балла):
      • Создать документацию с помощью Sphinx или MkDocs
      • Создать руководство по развертыванию
      • Настроить автоматическую генерацию документации
      • Создать примеры использования
    2. Публикация в Git Pages (3 балла):
      • Настроить GitHub Actions для автоматической публикации
      • Создать сайт с документацией на Git Pages
      • Настроить автоматическое обновление при изменениях
    3. Отчеты об экспериментах (2 балла):
      • Создать отчеты об экспериментах в формате Markdown
      • Добавить графики и визуализации результатов
      • Создать сравнительные таблицы экспериментов
      • Настроить автоматическую генерацию отчетов
    4. Воспроизводимость (1 балл):
      • Создать инструкции по воспроизведению
      • Создать README с полным описанием
      • Настроить автоматическую сборку документации

    Критерии оценки

    • Отлично (8 баллов): Полная документация, качественные отчеты
    • Хорошо (6-7 баллов): Хорошая документация, базовые отчеты
    • Удовлетворительно (4-5 баллов): Базовая документация
    • Неудовлетворительно (0-3 балла): Требования не выполнены

    Ход работы

    1. Выбор инструмента

    Для создания технической документации был выбран MkDocs.

    Причины выбора: - Markdown-native: Проект уже содержит множество отчетов в формате Markdown, которые MkDocs поддерживает "из коробки". - Простота настройки: Конфигурация осуществляется через один YAML-файл. - Современный интерфейс: Использование темы material обеспечивает отличный UX и адаптивность. - Docs-as-Code: Идеально вписывается в CI/CD процессы и философию хранения документации рядом с кодом. - Экосистема: Наличие плагина mkdocstrings позволяет легко генерировать документацию из Python docstrings.

    2. Подключение и настройка

    Процесс установки: 1. В pyproject.toml были добавлены зависимости в секцию dev: - mkdocs: основной движок. - mkdocs-material: современная тема оформления. - mkdocstrings[python]: плагин для автоматической генерации API-документации из кода. 2. Установка выполнена через uv: bash uv sync --extra dev

    Конфигурация: Создан файл mkdocs.yml в корне проекта со следующими настройками: - Использование темы material. - Настройка плагина mkdocstrings для поиска исходного кода в папке src. - Определение структуры навигации (nav), включающей существующие отчеты и новые страницы API Reference.

    Автоматизация API: Созданы файлы в docs/api/, использующие синтаксис ::: module.name для автоматического извлечения документации из Python-модулей.

    3. Локальный запуск

    Для проверки документации локально используется команда:

    uv run mkdocs serve

    Это запускает локальный сервер с поддержкой Live Reload по адресу http://127.0.0.1:8000.

    4. Публикация в GitHub Pages

    Для автоматизации публикации настроен CI/CD пайплайн через GitHub Actions.

    Настройка GitHub Actions: Создан файл docs.yml со следующим сценарием: 1. Триггер: Запуск при пуше в ветки master и feature/hw6_autodocs (для тестирования), если изменились файлы в docs/, mkdocs.yml или зависимости. 2. Окружение: Используется ubuntu-latest с установкой uv. 3. Сборка и деплой: - Установка зависимостей через uv sync --extra dev. - Выполнение команды mkdocs gh-deploy, которая собирает статический сайт и пушит его в ветку gh-pages.

    Результат: Теперь при каждом обновлении документации в репозитории, сайт на GitHub Pages будет обновляться автоматически.

    5. Отчеты об экспериментах и воспроизводимость

    Замечание по экспериментам: В рамках данного проекта основное отслеживание экспериментов (метрики, параметры, графики) реализовано через ClearML (см. HW5). Дублирование этого функционала в виде статических Markdown-отчетов является избыточным, так как ClearML предоставляет динамические дашборды, сравнение моделей и версионирование артефактов в реальном времени.

    Тем не менее, для выполнения требований: - Создана сводная таблица лучших результатов в hw_6_autodocs/experiments.md. - Добавлены инструкции по воспроизведению в hw_6_autodocs/reproduction.md. - Настроена автоматическая сборка всей документации (включая отчеты) через GitHub Actions.
